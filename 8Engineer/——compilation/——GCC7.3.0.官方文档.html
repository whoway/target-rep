                            # ⭐️14.控制流图『Control Flow Graph』

                            A control flow graph (CFG) is a data structure built on top of the intermediate code representation『中间代码表示』 (the ==RTL== or ==GIMPLE== instruction stream) abstracting the control flow behavior of a function that is being compiled.

                            控制流图 (CFG) 是一种建立在中间代码表示（RTL 或 GIMPLE 指令流）之上的数据结构，它抽象了正在编译的函数的控制流行为。

                            CFG 是一个有向图，其中顶点表示基本块，边表示控制流从一个基本块到另一个基本块的可能转移。 用于表示控制流图的数据结构在==basic-block.h==中定义。

                            14.1.基本块『Basic Blocks』

                            • whoway：==论文中经常出现==
                            • 基本块是只有一个入口点和一个出口的『straight-line』直线（性）代码序列
                              • 在 GCC 中，基本块使用==basic_block==数据类型表示
                            basic_block bb;

FOR_EACH_BB (bb)
{
    gphi_iterator pi;
    gimple_stmt_iterator si;

    for (pi = gsi_start_phis (bb); !gsi_end_p (pi); gsi_next (&pi))
    {
        gphi *phi = pi.phi ();
        print_gimple_stmt (dump_file, phi, 0, TDF_SLIM);
    }

    for (si = gsi_start_bb (bb); !gsi_end_p (si); gsi_next (&si))
    {
        gimple stmt = gsi_stmt (si);
        print_gimple_stmt (dump_file, stmt, 0, TDF_SLIM);
    }
}
