                                                  # 第1讲 引论

                                                  # 1.1 什么是编译程序

                                                  • 翻译程序(Translator)：把某一种语言程序(称为源语言程序)等价地转换 成另一种语言程序(称为目标语言程序)的程序
                                                  • 编译程序(Compiler)：把某一种高级语言程序等价地转换成另一种 低级语言程序(如汇编语言或机器语言程序)的程序
                                                    • 诊断编译程序(Diagnostic Compiler)
                                                    • 优化编译程序(Optimizing Compiler)
                                                    • 交叉编译程序(Cross Compiler)
                                                    • 可变目标编译程序(Retargetable Compiler)
                                                  • 解释程序(Interpreter)：把源语言写的源程序作为输入，但不产生目标 程序，而是边解释边执行源程序

                                                  # 1.2 为什么要学习编译原理

                                                  • 理解计算系统
                                                  • 设计计算系统
                                                  • 训练计算思维(Computational Thinking)
                                                    • 方法论的提取：Jeannette M. Wing, Computational Thinking (opens new window), Communications of ACM, Vol.49, No.3, 2006, pp.33-35.
                                                  包括一系列广泛的计算机科学的思维方法
 抽象(Abstraction)
 自动化
 问题分解
 递归
 权衡
 保护、冗余、容错、纠错和恢复
 利用启发式推理来寻求解答
 在不确定情况下的规划、学习和调度
 ...

                                                  • 遍(pass)
                                                  •  所谓"遍"， 就是对源程序或源程序的中间表示 从头到尾扫描一次
                                                  •  阶段与遍是不同的概念
                                                    •  一遍可以由若干段组成
                                                    •  一个阶段也可以分若干遍来完成
