    # 坑-自然连接和内连接有什么区别

    自然连接natural join

    aaa AS A natural join bbb AS B;
    1

    自然连接是基于两个表之间的共同列进行匹配的一种连接方式，它不是内连接（INNER JOIN）的一种。

    在自然连接中，MySQL会自动查找两个表中具有相同名称的列，并将它们作为连接条件。因此，自然连接不需要指定连接条件。而在内连接中，我们需要显式地指定要连接的列和连接条件。

    虽然自然连接和内连接都会返回匹配的行，但它们的区别在于，自然连接除了返回匹配的行，还会返回两个表中所有其他非重复的列。而内连接只返回符合连接条件的匹配行。

    需要注意的是，使用自然连接可能会造成一些潜在的问题。例如，如果两个表中具有相同名称的列包含了不同的数据类型，那么自然连接可能会出现数据类型不匹配的错误。此外，在具有多个匹配列的情况下，自然连接可能会导致意外的结果集。所以，在实际应用中，我们需要仔细考虑是否使用自然连接，或者选择使用其他类型的连接来实现我们的需求。

    # 3.3.截止Mysql8.0不支持-全连接

    截止当前最新版本 8.0.19，MySQL 尚未支持 Full Join（全外连接），但我们可以使用其它方式实现 Full Join 的效果。

    理论上，全外连接是左外连接和右外连接的组合。完整的外部连接包括联接表中的所有行，无论另一个表是否具有匹配的行。

    SELECT student.number, student.name, score.subject, score.score
from 
	student
full join 
	score 
on 
	student.number=score.number;

    # 4.组合查询-union和union all

    ​ 组合查询，使用 UNION 来组合两个查询，如果第一个查询返回 M 行，第二个查询返回 N 行，那么组合查询的结果一般为M+N 行

    • 1、每个查询「必须」包含相同的列、表达式和聚集函数。
    • 2、union默认会去除相同行」，如果需要保留相同行，使用 union all
    • 3、只能包含一个 ORDER BY 子句，并且必须位于语句的最后

    # [难点]MYSQL取表中分数排名前10%的学生信息

    WITH ranked_scores AS (
  SELECT name, subject, score, 
         ROW_NUMBER() OVER (PARTITION BY subject ORDER BY score ASC) AS myrank  
  FROM scores
)

SELECT name, subject, score
FROM ranked_scores
WHERE myrank <= CEIL((SELECT COUNT(1) FROM scores) * 0.1);
