    # 1.智能指针4个『C++11』

    ​ 智能指针（Smart pointer） 是一种pointer-like class『行为像指针的class』 。智能指针是C++提供给我们的一种工具，用来管理对象的一个生命周期的，简单来说就是来帮我们管理动态内存（也就是heap）的

    C++有个规定，除非如果你是智能指针，否则始终用深拷贝，vector为深拷贝

    C++中所有的对象都是深拷贝，只有shared_ptr和weak_ptr是浅拷贝，然后unique_ptr则是直接禁止拷贝这样的。

    在C语言中，他们有特定的算法来解决循环引用

    • 但在C++中，shared_ptr是解决不了，循环引用的。我们需要自己手动把其中1个需要的shared_ptr改为weak_ptr

    # 三五法则-让我意识到设计自己的class的时候，深浅拷贝的重要性

    • Java的话，这时候所以说他们索性简化说一切非基础类型的对象都是浅拷贝，然后都有1个引用计数，就不管你有没有自己用shared_ptr，它始终都是有引用计数的，然后引用计数它为了解决循环引用，它又用了垃圾回收机制自动管理。
    • 所以说Java他们这些管资源的语言呢，他们就喜欢把所有非基础都认为是一个shared pointer

    而C++是因为系统级的这些原因，才发展出三五法则那些东西。所以说我们如果想学好高性能计算的话，一定要学C++

    # 1.1. unique_ptr（独享指针）(RAII)

    • 学这个，一定要回move语句，std::move

    替换auto_ptr

    前置知识点：C++11引入的术语，右值引用，move语义

    特点：

    • unique_ptr 是 C++11 才开始提供的类型，是一种在异常时可以帮助避免资源泄漏的智能指针。
    • 采用l『『独占式拥有』』，意味着可以确保一个对象和其相应的资源同一时间只被一个 pointer 拥有。
    • 一旦拥有着被销毁或编程 empty，或开始拥有另一个对象，先前拥有的那个对象就会被销毁，其任何相应资源亦会被释放。
    unique_ptr auto_ptr
    C++11引入 C++98引入，C++11弃用
    1、无拷贝赋值语义2、但实现了move 语义； 1、可以赋值拷贝 2、复制拷贝后所有权转移
    unique_ptr 可以管理数组（析构调用 delete[] ） auto_ptr 对象不能管理数组（析构调用 delete）

    # 1.2.shared_ptr（共享指针）

    • 面试的时候读：share指针

    设计思想：实现：共享式拥有（shared ownership）概念

    应用场景举例：程序需要在多个对象间『共享数据』（或者叫：允许多个对象共享相同的状态）

    • 参考自《C++ primer》

    缺点：

    shared_ptr指针要是碰到，两个对象互相指向，导致“循环引用”会导致“内存泄漏”

    特点：

    • 1、支持定制型删除器（custom deleter）
    • 2、可防范 Cross-DLL 问题（对象在动态链接库（DLL）中被 new 创建，却在另一个 DLL 内被 delete 销毁）
    • 3、自动解除互斥锁
    • 4、线程不安全
    线程安全分为两个方面：
    shared_ptr智能指针对象中的引用计数是多个智能指针对象共享的，两个线程中智能指针的引用计数同时进行++或者 - -操作，在为加锁的情况下，会导致计数混乱，这样有可能造成资源泄漏或者程序奔溃的问题
    而且++和- -操作本身也不是原子的。智能指针管理的对象保存在堆上，两个线程同时去访问，也会导致线程安全问题。

    # 1.3. weak_ptr（弱指针）有失效检测

    为了解决“循环引用”导致的“内存泄漏”，引入了weak_ptr指针。

    • 《C++ Primer》一书中说它一个伴随类，它是一种弱引用，指向shared_ptr所管理的对象。

    特点：将一个weak_ptr绑定到一个shared_ptr不会改变shared_ptr的引用计数！！

    • 一旦最后1个指向对象的shared_ptr被销毁，对象就会被释放，即时有weak_ptr指向对象，对象也还是会被释放！！
    • 因此，weak_ptr的名字抓住了这种智能指针"弱"共享对象的特点！！

    参考：

    • weak_ptr 允许你共享但不拥有某对象『『好像，你没有权利拥有中科院文凭，但是你有权利观察某个学生的中科院文凭上的学号，但是观察不代表什么』』，一旦最末一个拥有该对象的智能指针失去了所有权，任何 weak_ptr 都会自动成空（empty）。因此，在 default 和 copy 构造函数之外，weak_ptr 只提供 “接受一个 shared_ptr” 的构造函数。
    • 可打破环状引用（cycles of references，两个其实已经没有被使用的对象彼此互指，使之看似还在 “被使用” 的状态）的问题

    # 1.4 auto_ptr（弃用的）C++98标准❌

    • C++98的时候的方案，C++11已抛弃
      • 采用所有权模式
      • 被 c++11 弃用，原因是缺乏语言特性如 “针对构造和赋值” 的 std::move 语义，以及其他瑕疵

    # 1.5.当智能指针-作为class的成员变量

    如果你的class里面有智能指针的话，那你可能这个类就变成1个浅拷贝的类了

    要么你的这个类里面有unique pointer，因为unique pointer他是禁止拷贝的。所以你这个类也会变成禁止拷贝的，就是呃他是会导致拷贝被删除，但移动还是有的。

    什么时候用什么类型的智能指针的时候，

    • weak_ptr和shared_ptr一起用
    • 原始指针和unique_ptr一起用

    因为weak_ptr他只能用shared_ptr

    而unique_ptr是不能生成1个weak_ptr的，unique只能生成1个原始指针，而原始指针是没有失效检测的。

    第5点建议的原因：

    1。一方面有unique_ptr禁止拷贝，可能就觉得很难用，然后原始指针又不能提供失效检测。

    所以share_ptr和weak_ptr的组合，所以这2可能是更安全，但是他是有性能损耗的。

    2、解决C++的share_ptr的循环引用，可以采用2个方案

    • shared_ptr和weak_ptr
    • shared_ptr和原始指针（缺点是没有失效检测）

    # 2.C++强制类型转换（Type Cast）

    # 背景

    # 场景1

    ​ C语言由于某个强制类型转换出错了，我们需要查找所有代码中相应的强制类型转换，那么会很难寻找。比如：

    double num=6.3131;
int solve=(int)num;
    1
    2

    ​ 原因：想要查找C语言风格的类型转换我们需要查找（这样的括号，但是函数也有啊
    ​ 解决：C++中设计了4种类型转换，而且给他们每个后面都有单词cast，对grep友好，我们快速定位类型转换，直接用Ctrl+F去寻找cast就好了

    # 场景2

    ​ 原因：由于C++中用基类和子类的指针，实现了多态。有时候，有些不熟悉多态的编码人员，可能会进行强制类型转换，导致项目崩溃。
    ​ 解决：C++中针对这些，设计了比如dynamic_cast类型转换

    # C++中4种类型转换如下

    ​ static_cast（静态类型转换）：静态类型转换，编译的时c++编译器会做类型检查，基本类型能转换但是不能转换指针类型

    ​ reinterpreter_cast（重新解释类型转换，interpreter，v.诠释，说明），若不同类型之间，进行强制类型转换，用reinterpret_cast进行重新解释

    ​ dynamic_cast（动态类型转换），C++中重要的，安全的基类和子类之间转换，运行时类型检查

    ​ const_cast（常量类型准换），去除变量的只读属性。

    # 一般结论

    • 1.C语言中能隐式类型转换的，在c++中可用static_cast进行类型转换。因C++编译器在编译检查一般都能通过；
    • 2.C语言中不能隐式类型转换的，在c++中可以用reinterpret_cast进行强行类型解释。
    • 3.总结：static_cast和reinterpret_cast基本上把C语言中的 强制类型转换给覆盖，但是reinterpret_cast很难保证移植性。

int main()

    //[Warning] deprecated conversion from string constant to 'char*' [-Wwrite-strings]

    # 7.总结

    1）编码人员要清楚的知道: 要转的变量，类型转换前是什么类型，类型转换后是什么类型。转换后有什么后果。

    2）一般情况下，不建议进行类型转换；避免进行类型转换。

    比如上面的“把形参的只读属性去掉”，我在C语言编程的时候，本来就是怕别人写的模块修改我传入的参数才用的const，但是这种类型转换会破坏我设计的初衷！
